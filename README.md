# Pizza Sales Analysis (SQL Project)

## 📌 Overview
This project analyzes a pizza sales dataset using SQL to extract key business insights such as revenue, customer behavior, and product performance.

The goal is to simulate a real-world business scenario, transforming raw transactional data into actionable insights through well-structured queries and KPIs.

## 🎯 Objectives
- Calculate key performance indicators (KPIs)
- Analyze sales trends (daily and monthly)
- Identify best and worst performing products
- Understand customer ordering patterns
- Support data-driven decision making

---

## 🛠️ Tools & Technologies
- SQL (SQL Server)
- Data Analysis
- Data Visualization (Dashboard)

---

## 📈 Key Metrics (KPIs)
- Total Revenue
- Average Order Value (AOV)
- Total Pizzas Sold
- Total Orders
- Average Pizzas per Order

---

# Analysis Performed

### 1. Sales Trends
- Daily order trends (identify busiest days)
- Monthly trends (seasonality insights)

### 2. Product Performance
- Top 5 and Bottom 5 pizzas by:
  - Revenue
  - Quantity sold
  - Number of orders

### 3. Sales Distribution
- % of revenue by pizza category
- % of revenue by pizza size

### 4. Category Analysis
- Total pizzas sold by category
- Identification of high-performing segments

---

## 🔍 Key Insights
- Weekends (Friday/Saturday) have the highest order volume
- Peak sales months: July and January
- Classic category drives the most revenue and orders
- Large size pizzas are the most popular
- A small group of products generates a large share of revenue (Pareto effect)
